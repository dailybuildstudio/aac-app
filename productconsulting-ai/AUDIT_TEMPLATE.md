# AI Product Audit — Deliverable Template

*This is the document you fill out and deliver for every $3,000 AI Product Audit. It's your core IP — the more repeatable and sharp it is, the faster you deliver and the easier you sell. Keep client-facing copy clean; delete these italic notes before sending.*

---

## How to use this template

1. **Run it in ~2 weeks, ~10–12 focused hours.** Don't gold-plate — fixed scope is the product.
2. Fill every `[bracketed]` field. Cut sections that don't apply.
3. Deliver as a clean PDF + a 45-minute walkthrough call (the call is where the upsell happens).
4. The deliverable should make the **next engagement obvious** without being a hard sell.

**Scope boundaries (state these up front, every time):**
- ✅ In scope: review of the existing product, AI opportunity identification, ROI/effort scoring, a prioritized roadmap, and one detailed "build-first" recommendation.
- ❌ Out of scope: writing code, building the feature, data engineering, vendor procurement, ongoing involvement. (Those are the Sprint / retainer / fractional engagements.)

## 3. The opportunity scoring model
*Your signature framework. Score every candidate the same way so prioritization is defensible, not vibes.*

Each opportunity scored 1–5 on:

| Dimension | What it measures |
|-----------|------------------|
| **User value** | How much this improves the user's core job / outcome |
| **Business impact** | How directly it moves a metric that matters (revenue, retention, cost) |
| **Feasibility** | How realistic given their data, team, and tools (5 = easy) |
| **Time to value** | How fast it can ship and show results (5 = weeks, not quarters) |
| **Defensibility / strategic fit** | Whether it compounds advantage vs. being a commodity bolt-on |

**Priority score = (User value + Business impact + Time to value) − (5 − Feasibility).**
*High value + fast + feasible rises to the top. Use judgment; the number starts the conversation, it doesn't end it.*
